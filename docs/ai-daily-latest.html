How we build evals for Deep Agents

本文介绍了在开源的 Deep Agents 框架中，如何通过精心设计的 评估（evals） 来塑造和提升代理行为。核心要点包括： 明确目标行为：先确定生产环境中关键的行为需求，如跨文件检索或连续调用多个工具。 针对性评估：针对每个行为研发可验证的评估，并在评估代码中加入文档字符串说明其测量方式，确保自解释性。 标签与分组：为评估添加如 tool_use 等标签，以便批量运行和管理。 追踪与分析：所有评估运行都记录在 LangSmith 项目中，团队成员可随时审查失败模式并更新评估覆盖。 通过上述流程，避免盲目堆砌大量评估，确保每个评估都真实反映生产需求，从而使代理更准确、可靠。

How Middleware Lets You Customize Your Agent Harness

中间件如何让您定制代理框架 文章介绍了代理框架（agent harness）的概念及其定制方式。核心要点包括： 代理框架定义： 将大型语言模型（LLM）与环境、数据、记忆和工具连接的系统，核心是 LLM 循环调用工具。 定制需求： 不同业务场景需要在指令、工具或核心循环中加入自定义步骤，如预处理或结果检查。 AgentMiddleware 作用： 提供一组 hook（如 before_agent、after_agent、before_tool、after_tool），允许在循环的每个阶段插入自定义逻辑，实现灵活而安全的定制。 实现方式： 基于 LangChain 的 create_agent 抽象，开发者可在保持核心框架不变的前提下，通过中间件扩展功能。 通过使用 AgentMiddleware，开发者能够在不重写底层循环的情况下，实现针对特定应用的深度定制。
