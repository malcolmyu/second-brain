# 小马技能开发路线图

基于于明昊深度用户画像，设计3个核心技能。

---

## Skill 1: 深度调研报告生成器 (research-report)

### 用户痛点
- 需要快速产出结构化调研报告
- 期望交付物是可部署的交互式网页，不是聊天记录
- 关注"为什么"多于"怎么做"

### 功能定义
**输入**: 一句话指令，如"帮我做一个字节AI布局的深度调研"
**输出**: 完整的交互式 HTML 报告，包含：
- 执行摘要（核心洞察）
- 背景与方法论
- 多维度分析（按主题分层）
- 数据可视化（图表）
- 结论与建议

### 技术实现
- 使用 `web_search` 获取多源信息
- 使用 `autoglm-deepresearch` 进行深度分析
- 使用 `frontend-design` skill 生成交互式 HTML
- 输出到 `workspace/reports/YYYY-MM-DD-{topic}.html`

### 对标林克
林克30分钟生成字节AI布局报告 → 小马60分钟生成同等质量报告（第一版）

## Skill 2: 快手工作流集成 (kuaishou-workflow)

### 用户痛点
- 需要在多个快手内部工具间切换
- Team、Kwaibi、Feishu、Docs 数据分散
- 期望一句话完成跨工具操作

### 功能定义
**场景1 - 每日工作摘要**: 
"总结我昨天在 Team 的进展和今天的待办"
→ 读取 Team 任务 + Kwaibi 数据 + 生成摘要

**场景2 - 周报生成**:
"生成本周周报"
→ 汇总本周 Team 任务完成状态 + 关键产出

**场景3 - 智能提醒**:
"提醒我即将到期的任务"
→ 扫描 Team 任务，推送即将到期的到 Feishu

### 技术实现
- 集成 `team-ops` skill
- 集成 `kwaibi-ops` skill  
- 集成 `feishu-doc` skill
- 使用 `message` 工具推送通知

### 快捷键设计
- `/daily` - 每日摘要
- `/weekly` - 周报生成
- `/remind` - 到期提醒

---

## Skill 3: 自主进化心跳 (self-evolution)

### 用户痛点
- 希望AI分身能自主学习、持续进化
- 期望每天早上有"昨日回顾 + 今日建议"
- 关注长期价值积累

### 功能定义
**每天 9:00 自动执行**:
1. **昨日回顾**: 读取昨日 OpenClaw 会话历史
2. **模式识别**: 提取关键决策、重复任务、新学技能
3. **知识归档**: 将洞察写入 memory/YYYY-MM-DD-insights.md
4. **今日建议**: 基于历史模式，生成今日工作建议
5. **推送通知**: 发送到 Feishu（简洁版摘要）

### 技术实现
- 使用 `sessions_history` 读取会话历史
- 使用 `memory_store` 归档洞察
- 使用 `cron` 配置定时任务
- 使用 `message` 推送 Feishu

### 输出格式
```
📊 小马昨日洞察（2026-03-14）

🎯 关键决策:
- 启动数字分身构建项目（优先级：高）

📚 新学技能:
- 使用 subagent 并行开发

💡 今日建议:
- 完成架构设计评审
- 开始第一个技能开发
```
