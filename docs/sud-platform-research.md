# SUD 游戏化互动平台 SDK 接入与支付对接调研

## 一、平台概述

**SUD (Super Unified Development)** 是一个游戏化互动平台，提供：
- 小游戏 SDK 接入
- 语音/视频互动能力
- 支付系统
- 社交功能

---

## 二、SDK 接入方式

### 2.1 接入流程（通用模式）

```
1. 注册开发者账号
   ↓
2. 创建应用，获取 AppID 和 AppKey
   ↓
3. 下载对应平台 SDK（iOS/Android/Web/Unity）
   ↓
4. 集成 SDK 到项目
   ↓
5. 配置服务端回调 URL
   ↓
6. 测试环境联调
   ↓
7. 上线发布
```

### 2.2 SDK 核心功能模块

| 模块 | 功能描述 |
|------|----------|
| **Core** | 初始化、用户认证、基础配置 |
| **Game** | 游戏加载、游戏状态管理、分数上报 |
| **Voice** | 实时语音、语音房间管理 |
| **Payment** | 支付、订单管理、回调处理 |
| **Social** | 好友、排行榜、消息 |

### 2.3 接入示例（Web）

```javascript
// 初始化
SUD.init({
  appId: 'YOUR_APP_ID',
  appKey: 'YOUR_APP_KEY',
  environment: 'production' // or 'sandbox'
});

// 用户登录
SUD.auth.login({
  userId: 'user_123',
  token: 'USER_AUTH_TOKEN'
});

// 加载游戏
SUD.game.load({
  gameId: 'GAME_ID',
  container: document.getElementById('game-container'),
  onReady: () => console.log('Game ready'),
  onScore: (score) => console.log('Score:', score)
});
```

## 三、支付系统与内部金币体系对接

### 3.1 核心问题

**用户需求**：将 SUD 的付费系统接入公司内部的金币体系。

### 3.2 对接方案

#### 方案 A：服务端代理模式（推荐）

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   客户端     │────▶│  你的服务端  │────▶│  SUD 服务端  │
│ (iOS/Android)│     │             │     │             │
└─────────────┘     └──────┬──────┘     └─────────────┘
                           │
                           ▼
                    ┌─────────────┐
                    │ 公司金币系统 │
                    └─────────────┘
```

**流程**：
1. 用户在前端选择购买 SUD 道具/金币
2. 请求发送到你的服务端
3. 服务端先扣除用户的公司金币
4. 服务端调用 SUD API 完成购买
5. SUD 回调通知购买成功
6. 服务端通知客户端刷新余额

**优点**：
- 完全控制支付流程
- 可以加入公司风控逻辑
- 统一账单管理

**缺点**：
- 需要维护服务端逻辑
- 增加延迟

#### 方案 B：客户端中转模式

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   客户端     │────▶│  你的服务端  │     │  SUD SDK    │
│             │     │ (金币扣除)   │────▶│ (道具发放)  │
└─────────────┘     └─────────────┘     └─────────────┘
```

**流程**：
1. 客户端调用公司金币系统扣除金币
2. 扣除成功后调用 SUD SDK 购买
3. SUD 回调确认

**优点**：
- 实现简单
- 延迟低

**缺点**：
- 容易被破解/篡改
- 难以保证事务一致性

#### 方案 C：虚拟账户映射模式

```
公司金币  ←→  SUD 金币 (固定汇率)
   │              │
   └──────┬───────┘
          │
    自动兑换服务
```

**流程**：
1. 用户充值时，公司金币和 SUD 金币同步充值
2. 或者设置自动兑换比例
3. 使用 SUD 原生支付流程

## 四、服务端 API 对接

### 4.1 必要的 API 接口

```javascript
// 1. 创建订单（你的服务端）
POST /api/sud/order/create
{
  "userId": "user_123",
  "productId": "sud_coin_100",
  "companyCoinCost": 1000
}

// 2. 扣除公司金币后，调用 SUD 创建订单
POST https://api.sud.tech/v1/orders
Headers: {
  "Authorization": "Bearer SUD_ACCESS_TOKEN",
  "Content-Type": "application/json"
}
Body: {
  "appId": "YOUR_APP_ID",
  "userId": "user_123",
  "productId": "sud_coin_100",
  "orderId": "YOUR_UNIQUE_ORDER_ID",
  "notifyUrl": "https://your-api.com/sud/callback"
}

// 3. SUD 支付回调处理
POST /sud/callback
{
  "orderId": "ORDER_ID",
  "status": "success",
  "signature": "SIGNATURE"  // 需要验签
}
```

### 4.2 安全考虑

1. **签名验证**：所有回调必须验证签名
2. **幂等处理**：同一订单多次回调只处理一次
3. **对账机制**：定期与 SUD 对账
4. **事务回滚**：公司金币扣除成功但 SUD 购买失败时，需要回滚
